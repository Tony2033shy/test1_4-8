最長公共子序列 (LCS)

動態規劃 (Dynamic Programming) 的經典演算法解析

什麼是「子序列」(Subsequence)？

子序列是從原始字串中刪除部分字元（或不刪除），並且不改變剩餘字元相對順序所形成的新字串。

它與「子字串」(Substring) 的最大差別在於，子序列中的字元在原字串中不需要連續。


例如字串 X = "ABCDE"

  • "ACE" 是子序列（保留 A, C, E）。
  • "CAE" 不是子序列（C 不能在 A 前面）。

LCS 的定義與重要性

核心任務

給定兩個字串 X 和 Y，找出它們共同擁有的最長子序列。


例如：
X = "ABCDGH"
Y = "AEDFHR"

它們的 LCS 是 "ADH"，長度為 3。

為什麼重要？

LCS 是衡量兩個序列「相似度」的標準數學模型。當兩個序列的組成越相似，它們的最長公共子序列長度就會越長。

這個單純的概念，構成了現代許多軟體與生物科技底層的基礎演算法。

解決方案：暴力破解 vs. 動態規劃

  • 暴力破解法 (Brute Force)：
    枚舉字串 X 的所有可能子序列，再逐一檢查是否也是 Y 的子序列。若 X 長度為 m，子序列數量高達 2m 種。時間複雜度為 O(n·2m)，當字串稍長時，計算量將呈指數級爆炸。
  • 動態規劃 (Dynamic Programming)：
    利用 LCS 問題具備的「最佳子結構」與「重疊子問題」特性，將大問題拆解成小問題，並把計算過的結果儲存在表格中，避免重複計算。時間複雜度可大幅降至 O(m·n)。

大哉問：怎麼把大問題拆小？ (字元相同時)

假設我們正在比對兩個字串的「最後一個字元」：

X = "ABC"
Y = "ADC"
  • 直覺思考：既然最後一個字元都是 'C'，那麼這個 'C' 一定會包含在我們的最長公共子序列中！
  • 問題縮小：我們只要把 'C' 拿掉，去尋找前面剩餘部分 "AB" 和 "AD" 的 LCS 長度，然後再加上 1 (也就是這個 'C') 即可。

大哉問：怎麼把大問題拆小？ (字元不同時)

如果最後一個字元「不一樣」呢？

X = "ABCD"
Y = "ABCE"
  • 直覺思考：因為 'D' 和 'E' 不同，它們不可能「同時」成為 LCS 的最後一個字元。
  • 問題分岔：既然不可能同時出現，那我們就分兩條路走，看看哪條路找到的序列比較長：

    👉 路線 1：丟掉 X 的尾巴，拿 "ABC" 和 "ABCE" 繼續比。
    👉 路線 2：丟掉 Y 的尾巴，拿 "ABCD" 和 "ABC" 繼續比。

動態規劃：狀態轉移方程式

經過剛才的推導，我們將概念化為數學式。
令 dp[i][j] 代表字串 X[0..i-1] 與 Y[0..j-1] 的 LCS 長度：

dp[i][j] = { 0 if i=0 or j=0 dp[i-1][j-1]+1 if Xi=Yj (字元相同) max(dp[i-1][j],dp[i][j-1]) if Xi≠Yj (字元不同)

程式碼實作 (Python)

我們將剛剛的公式直接翻譯成程式碼：

  • 建立一個 (m+1) × (n+1) 的二維陣列，預設為 0。
  • 使用雙層迴圈依序掃描兩個字串。
  • 根據字元是否相等，執行對應的狀態轉移。
  • 最後回傳表格最右下角的值 dp[m][n]。
def get_lcs_length(X: str, Y: str) -> int:
    m, n = len(X), len(Y)
    # 初始化 dp 陣列 (多開一格放 0)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if X[i-1] == Y[j-1]:
                # 字元相同：左上角 + 1
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                # 字元不同：取上方或左方最大值
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
                
    return dp[m][n]

程式碼實作 (C++)

在 C++ 中，我們推薦使用 STL 的 vector 來宣告二維陣列。

這種寫法比起傳統的指標宣告更安全，也能在宣告的同時自動把所有格子初始化為 0，省去手動寫迴圈填 0 的麻煩。

int getLCSLength(string X, string Y) {
    int m = X.length(), n = Y.length();
    
    // 建立 (m+1)*(n+1) 的 vector，初始為 0
    vector<vector<int>> dp(m+1, vector<int>(n+1, 0));
    
    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (X[i-1] == Y[j-1]) {
                dp[i][j] = dp[i-1][j-1] + 1;
            } else {
                dp[i][j] = max(dp[i-1][j], dp[i][j-1]);
            }
        }
    }
    return dp[m][n];
}

演算法複雜度分析

O(M×N)
時間複雜度

極致的空間優化

基本的 DP 解法需填滿一個 M 乘 N 的陣列，故空間複雜度亦為 O(M×N)。

但在計算時，我們發現轉移方程式永遠只會依賴「上一列」與「當前列」。利用「滾動陣列 (Rolling Array)」技巧，只保留兩列空間交替使用，可將空間複雜度大幅降至：

O(min(M,N))
